# 宝丰一高校园频道 · 项目汇总

> 汇总日期: 2026-07-14 | 范围: 全项目（前端 + 后端 + 部署 + 文档）

---

## 一、项目是什么

宝丰一高校园社交平台。核心三块：

1. **登录页** —— 学校夜景照片 + 流星雨动画 + 3D 轮播 + 毛玻璃弹窗卡片
2. **主聊天界面** —— QQ/Discord 风格三栏布局（频道列表 / 消息流 / 成员面板），Socket.io 实时聊天
3. **后台管理** —— 用户管理 / 帖子审核 / 数据看板

定位：班级 / 社团级别（~100 人），可扩展至全校。

---

## 二、技术栈

| 层   | 技术                                   | 说明                   |
| --- | ------------------------------------ | -------------------- |
| 前端  | 原生 HTML/CSS/JS + Socket.io 客户端       | 无框架依赖                |
| 后端  | Node.js 22 + Express 4 + Socket.io 4 | REST API + WebSocket |
| 数据库 | SQLite（sql.js WASM 版本）               | 整库常驻内存，定时写回          |
| 认证  | JWT + bcrypt（12 轮）                   | 登录返回 token           |
| 部署  | Docker + Nginx / CloudBase 云托管       | 已做容器化适配              |

## 六、已知问题 / 待办（按优先级）

### 🔴 P0（高，影响上线与合规）

1. **数据持久化风险**（体检 #2/#3）：SQLite 用 sql.js（WASM）整库常驻内存、每 5s 写回；上传文件落本地磁盘。**容器重启即丢数据、多实例扩容各持独立库 → 数据错乱**。→ 云托管必须挂持久卷（`PERSIST_DIR=/data`），且保持**单实例**。
2. **登录/注册无 API 级限流防爆破**（体检 #6）：可被暴力破解 / 刷接口。
3. **缺失 `prefers-reduced-motion` 可访问性支持**（UI 评审 P0）：流星、校徽呼吸光、轮播同时运动，前庭功能障碍用户可能不适。

### 🟡 P1（中）

1. `npm run seed` 指向不存在的 `seed.js`（实际是 `seed-auto.js`）→ 脚本报错（体检 #4）。
2. 前端 `app.js` 调用后端不存在的 `POST /api/messages/:id`（消息仅经 WS 发送）→ WS 断开时 404，在线时双发（体检 #5）。
3. 登录入口「Sign In / Sign Up」中英文混杂，与中文界面割裂 → 改为「登录 / 注册」（UI 评审 P0 级品牌一致性）。
4. 移动端触摸目标偏小（28px < 44px）、侧栏滑出缺遮罩（UI 评审 P1）。
5. 个人主页信息为占位内容（身份/频道/状态显示 `#`、`在线`），数据未绑定（UI 评审 P1）。

### 🟢 P2（低 / 加固）

1. CORS 全开 + Socket `origin:'*'`；`JWT_SECRET` 有硬编码兜底（部署必设环境变量）（体检 #7/#8）。
2. 无 helmet 安全头、注册无验证码（体检 #9）。
3. CSS ~1900 行未拆分；登录轮播图片未懒加载；校徽红框与紫色品牌色冲突（UI 评审 P2）。

## 八、一句话结论
