# 宝丰一高校园频道 · 功能导览视频设计

## 背景
第一版介绍视频（`video/promo/out/promo.mp4`，36s）定位为「品牌预告片」，只跑了核心循环（登录→频道→聊天→发动态），未覆盖用户实际网站上的大量功能。用户要求补拍：动效、各频道功能、隐藏校徽动画、校园介绍、技术支持、背景设置，并追加「消息图片」与「@提醒」。

经头脑风暴确认方向 **A：做一支更长的新视频（功能导览）**，把上述功能全拍进去，而非保留预告片或扩充成说明向。

## 方案选择
- **A（选定）：独立功能导览视频，~90s**。完整覆盖所有功能，拍法与 v1 不同——v1 是手搓暗色玻璃 UI 模拟，本片**登录真站点抓真面板截图/录交互**再进 Remotion 串联，校园介绍/背景设置等是真画面。
- B（未选）：保留 36s 预告片 + 另做功能详解。分工清晰但用户要一支搞定。
- C（未选）：把 36s 扩成 60s 塞功能。节奏变说明向，稀释电影感。

## 总体规格
- 时长 **~90s（2700 帧 @30fps）**，1080p。
- 工程 `video/tour/`（新建），复用 v1 的 `theme.ts` / `content.ts` / `GlobalFonts.tsx` / `Meteors.tsx`。
- 字体沿用 v1 子集字体 `noto-sc-400.woff2` / `noto-sc-700.woff2`（CJK 已子集化，渲染不糊）。
- BGM 沿用 v1 `bgm.mp3`（28% 音量循环）或换更贴合导览的曲子（待定）。
- 真截图手段：`admin@baofeng.campus` / `wzy200812` 登录 `bfgzlt.cc.cd`，Puppeteer/Playwright 抓 2x 图并脱敏；交互片段用逐帧截图序列（~100ms/帧）或录屏拼接。

## 新增功能处理
- **@提醒**：放在 S5 beat ②，演示「收到 @小宇」消息时高亮闪一下（前端 `msg-hot-highlight` 类效果，真前端可截）。

## 风险与对策
- **聊天实时层未部署** → 聊天 UI 可截，但消息需登录后手动预置 demo 数据（发几条带 @ / 图片的消息）再截；InsForge 登录态不稳时用 Playwright 替代 Puppeteer。
- **编码事故复发**（历史：GBK Mojibake 损坏 HTML/JS）→ 所有含中文文件用 UTF-8 工具读写；改完 `node --check` 验证 JS 语法再构建。
- **版本号不一致白屏**（历史）→ 构建/部署后强制刷新验证，必要时换全新版本号破缓存（本片为独立 `video/tour/`，不污染主站 `web_build/`）。

## 验收标准
- 整片 1080p@30fps，时长 88–92s，无黑帧/花屏/中文豆腐块。
- S2 校徽点击展开、S7 主题切换实时变两段为真实交互效果（非静态图硬切）。
- 字幕与分镜表一致；@提醒、消息图片两处新增 beats 清晰可辨。
- 渲染后独立视觉审查一遍（参照 v1 流程），修复后出最终 MP4。
